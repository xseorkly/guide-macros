Guide de démarrage
1

Ouvrir l'éditeur VBA

Appuyez sur Alt + F11 dans Excel. L'éditeur Visual Basic s'ouvre dans une nouvelle fenêtre.

2

Créer un module

Dans le menu de l'éditeur, cliquez sur Insertion → Module. Un module vide apparaît dans l'explorateur de projet à gauche.

3

Coller le code VBA

Cliquez dans la zone blanche du module, puis collez le code. Chaque macro commence par Sub NomMacro() et se termine par End Sub.

4

Tester la macro

Placez votre curseur à l'intérieur du code, puis appuyez sur F5 pour l'exécuter. Utilisez F8 pour exécuter ligne par ligne (débogage).

5

Enregistrer en .xlsm

Pour conserver les macros, sauvegardez en .xlsm (Classeur Excel avec macros). Le format .xlsx supprime silencieusement tout le code VBA.

6

Associer à un bouton

Onglet Développeur → Insérer → Bouton (contrôle de formulaire). Dessinez le bouton sur la feuille, puis associez-le à votre macro dans la boîte de dialogue.

🔧 Activer l'onglet Développeur

Fichier → Options → Personnaliser le ruban → cocher Développeur. Donne accès à l'éditeur VBA, aux macros et aux contrôles de formulaire.

🔐 Niveau de sécurité des macros

Développeur → Sécurité des macros → "Désactiver toutes les macros avec notification". Excel demandera confirmation à chaque ouverture d'un fichier avec macros.

📂 Emplacement approuvé

Fichier → Options → Centre de gestion de la confidentialité → Emplacements approuvés. Les fichiers de ce dossier s'ouvrent avec les macros activées automatiquement.

📚 Références VBA nécessaires

Dans l'éditeur VBA : Outils → Références. Pour Outlook, cochez Microsoft Outlook Object Library. Pour Word, cochez Microsoft Word Object Library.

💾 Classeur de macros personnelles

Lors de l'enregistrement d'une macro, choisir "Classeur de macros personnelles" pour que la macro soit disponible dans tous vos fichiers Excel.

🔁 Activation au démarrage

Pour les fichiers internes de confiance, l'option "Activer toutes les macros" peut être utilisée. Évitez-la pour des fichiers reçus par e-mail ou de sources externes.

⛔

Oublier de sauvegarder en .xlsm

Si vous sauvegardez en .xlsx, tout votre code VBA est silencieusement supprimé. Excel vous prévient mais beaucoup cliquent "Oui" sans lire le message.

Toujours choisir "Classeur Excel prenant en charge les macros (*.xlsm)" lors de l'enregistrement.
⚠️

Ne pas gérer les erreurs avec On Error

Une macro sans gestion d'erreur plante brutalement dès qu'une cellule est vide, un fichier absent ou un format inattendu — laissant parfois les données dans un état intermédiaire.

Ajouter On Error GoTo GestionErreur en début de macro et un bloc de traitement à la fin.
⛔

Utiliser Select et Activate inutilement

Les macros enregistrées utilisent .Select partout. C'est lent, fragile et ça plante si une autre feuille est active. Source d'erreur n°1 des débutants.

Référencer directement : Sheets("Data").Range("A1").Value = 10 sans passer par Select.
⚠️

Coder en dur les noms de feuilles et plages

Si vous écrivez Sheets("Rapport Janvier") et que l'onglet est renommé, la macro plante. Idem pour des plages fixes quand les données grandissent.

Utiliser des variables pour les noms et Cells(Rows.Count,"A").End(xlUp).Row pour les plages dynamiques.
⛔

Ne pas désactiver le rafraîchissement écran

Sans Application.ScreenUpdating = False, chaque action rafraîchit l'affichage. Une macro traitant 1 000 lignes peut prendre 60 s au lieu de 2 s.

Ajouter Application.ScreenUpdating = False au début et = True à la fin.
⚠️

Pas d'annulation possible (pas d'Undo)

Les macros VBA vident la pile d'annulation d'Excel. Une fois exécutée, Ctrl+Z ne permet pas de revenir en arrière.

Créer une copie de sauvegarde avant d'exécuter une macro destructive, ou demander confirmation avec MsgBox.
ℹ️

Oublier Option Explicit

Sans Option Explicit, une faute de frappe dans un nom de variable crée silencieusement une nouvelle variable vide. Le bug est quasi-invisible.

Ajouter Option Explicit en première ligne de chaque module pour forcer la déclaration des variables.
